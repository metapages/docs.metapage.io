File System

Each metapage has a scoped file system after the /fs/ part of the URL. This stores inputs and outputs from the data flow, metadata files such as images, and files you add or create. These files can be connected to other metapage workflows, allowing the outputs of one workflow to be consumed by another workflow.

The Files button shows all metapage files:

You can drag files to upload, delete, and rename files.

Add a file system view to your metapage​

You can create a metaframe that is a view of the metapage file system. The metaframe file system (fs) view is reactive: updates are real-time. Files are not automatically sent downstream unless you click on a file, or the File Selector view is chosen from the options, or Send Files Automatically is selected in the options.
