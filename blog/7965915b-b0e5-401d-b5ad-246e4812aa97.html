metapage URL schema

October 12, 2023 · One min read

The way URLs are constructed:

  • Need to be consistent with current and future planned behaviour
  • Simple to understand
  • Work with different methods of showing metapages
    • The official editor: app.metapage.io
    • The standalone viewer
    • Others
  • Allow arbitrary path names
    • E.g. users can name a metapage foo/bar/thing and it shows up as https://app.metapage.io/<user>/foo/bar/thing
    • Then path hierarchies can be show to users

The last part is important as (currently) arbitrary path names do not work as operations are also specified in the path: https://app.metapage.io/<user>/<name>/edit

Solution​

  • foo/bar/thing is a valid metapage name
  • Anything after /<user>/ is the metapage name
  • Operations use query parameters
    • ?view=edit, ?view=default, ?view=minimal or whatever
  • State is hash parameters
